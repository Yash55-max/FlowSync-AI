# Technical Requirements Document (TRD)

## 1. System Architecture

### Components:

* Frontend (React)
* Backend API (FastAPI)
* AI Module (Python)
* Realtime DB (Firebase)
* Maps Integration (Google Maps API)

## 2. Data Flow

1. Simulated crowd data generated
2. AI model processes density
3. Backend updates Firebase
4. Frontend listens and updates UI in real-time

## 3. Modules

### 3.1 Crowd Simulation Module

* Generate random crowd density data
* Grid-based mapping

### 3.2 AI Prediction Module

* Input: crowd density
* Output: congestion levels + predictions

### 3.3 Routing Engine

* Simple shortest path + avoid high-density nodes

### 3.4 Backend API

Endpoints:

* /heatmap
* /route
* /queue-time

### 3.5 Frontend

* Heatmap visualization
* Route display
* Alerts UI

## 4. Tech Stack

* React + Tailwind
* FastAPI
* Firebase Realtime DB
* Google Maps API
* Python (NumPy, OpenCV optional)

## 5. Database Schema (Firebase)

crowd_data:

* zone_id
* density_level
* timestamp

queues:

* stall_id
* wait_time

routes:

* start
* end
* path

## 6. AI Approach

* Rule-based + simple ML (regression or thresholding)
* Avoid heavy training due to time constraints

## 7. Security

* Firebase Auth (optional)
* No personal data storage

## 8. Performance

* Use WebSockets/Firebase listeners
* Optimize updates (only changed zones)

## 9. Testing

* Simulate multiple users
* Test routing accuracy
* Validate real-time updates

## 10. Deployment

* Frontend: Vercel
* Backend: Render
* Database: Firebase
